# API conventions

Shared patterns across Jaza Venus REST endpoints: pagination, document lifecycle, division scoping, and hypermedia-style linking.

---

## Base URL and versioning

| Environment | Base URL |
|-------------|----------|
| Local dev | `https://localhost:5001` |
| Production (Fly.io) | `https://jaza-venus.fly.dev` |
| SPA (same-origin) | `/api/*` (proxied to backend) |

All routes are prefixed with `/api`. OpenAPI document: `/openapi/v1.json` (Development).

There is no `/api/v1` path segment — version is declared in OpenAPI metadata only.

## Pagination

### List endpoints (`PagedRequest`)

| Query param | Default | Max | Notes |
|-------------|---------|-----|-------|
| `page` | 1 | — | 1-based index |
| `pageSize` | 20 | 50 | Capped by `PagedRequest.Normalized()` |
| `search` | — | — | Full-text where supported |
| `sort` | — | — | Entity-specific sort keys |

Response envelope `PagedResult<T>`:

```json
{
  "items": [ /* T[] */ ],
  "totalCount": 142,
  "page": 1,
  "pageSize": 20,
  "totalPages": 8
}
```

### Report endpoints (`ReportQueryParams`)

| Query param | Default | Max |
|-------------|---------|-----|
| `page` | 1 | — |
| `pageSize` | 100 | 500 |
| `from` / `to` | — | Date range filters |
| `division` | caller scope | Override for Developer/SuperAdmin |

Reports return `ReportQueryResult` with column metadata and row arrays.

## Document status

All transactional documents share `DocumentStatus`:

| Value | Name | Meaning |
|-------|------|---------|
| `0` | Draft | Editable; not yet affecting stock or AR |
| `10` | Posted | Immutable; stock/AR effects applied |
| `90` | Voided | Cancelled after posting (invoice void only) |

### Lifecycle rules

1. **Create** → status `Draft`
2. **Update** (`PUT`) → only while `Draft`
3. **Post** (`POST …/{id}/post`) → transitions to `Posted`; irreversible via update
4. **Void** (`POST …/{id}/void`) → invoice only; SuperAdmin; → `Voided`

Corrections after posting use reversal documents (returns, credit memos, adjustments).

Document DTOs include `status` as numeric enum value or string depending on serializer settings (JSON uses enum names by default in ASP.NET Core).

## Division scoping

Legacy `CompanyIDKu` maps to string division codes on documents and master data:

| Code | Description |
|------|-------------|
| `DISTRIBUTIONBDG` | Distribution Bandung |
| `DISTRIBUTIONCRB` | Distribution Cirebon |
| `TRADINGBDG` | Trading Bandung |
| `TRADINGCRB` | Trading Cirebon |

### Read scoping

- **Sales / Admin**: lists filtered to user's preference division (default `DISTRIBUTIONBDG`)
- **Developer / SuperAdmin**: all divisions unless `X-Division` header pins one

### Write scoping

- `IDivisionScopeService.RequireDivisionForWrite()` assigns division on create
- Admin/Sales without pinned division must send `X-Division` on mutating requests
- Cross-division reads return 403

### Preference storage

User division preference is stored in `user_preferences` and returned from `GET /api/auth/me`. Update via `PUT /api/auth/preferences` when the field is exposed on the DTO.

## Base linking

Documents reference related entities by **GUID foreign keys** plus optional denormalized display fields:

```json
{
  "id": "019dfb7d-1925-72ad-81ba-b7c5b34d6b7e",
  "number": "PO-2026-0042",
  "supplierId": "019dfb7d-0001-7000-8000-000000000001",
  "supplierName": "PT Supplier Jaya",
  "warehouseId": "019dfb7d-0002-7000-8000-000000000002",
  "warehouseCode": "WH-BDG-01"
}
```

There is no HATEOAS `_links` envelope. Clients construct URLs from known patterns:

| Relation | Pattern |
|----------|---------|
| Supplier detail | `GET /api/master/suppliers/{supplierId}` |
| PO from GRN | `GET /api/inbound/purchase-orders/{purchaseOrderId}` |
| Invoice from DO | `GET /api/invoices/{invoiceId}` |

Document numbers (`PO-…`, `SO-…`, `INV-…`) are unique per division and generated by `IDocumentNumberGenerator`.

## Request conventions

| Topic | Convention |
|-------|------------|
| Content-Type | `application/json` for bodies |
| IDs | UUID v7 (`guid`) in path and body |
| Dates | ISO 8601 UTC (`2026-07-10T09:30:00Z`) |
| Currency | ISO 4217 three-letter code (`IDR`) |
| Quantities | Decimal (supports fractional UOM) |
| Soft delete | Master data uses `IsDeleted`; omitted from lists |

## CSRF and auth headers

| Header | When |
|--------|------|
| `Authorization: Bearer {jwt}` | Mobile / API clients |
| `Cookie: jaza.auth=…` | Browser SPA |
| `X-XSRF-TOKEN` | Cookie clients on POST/PUT/PATCH/DELETE |
| `X-Division` | Pin division for writes or Developer list filter |
