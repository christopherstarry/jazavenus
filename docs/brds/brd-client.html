Dokumen Persetujuan Proyek
Mei 2026

Dokumen ini menjelaskan rencana proyek pembaruan aplikasi Jaza secara ringkas. Tujuannya agar Bapak/Ibu memahami apa yang akan kami kerjakan, kapan selesainya, dan apa manfaatnya sebelum proyek dimulai.

1 Ringkasan

Aplikasi Jaza saat ini berjalan di Windows XP dengan teknologi lama. Aplikasi ini tidak bisa diperbarui, tidak bisa dipasang di komputer baru, dan berisiko berhenti bekerja sewaktu-waktu.

Yang paling penting: aplikasi baru akan berjalan persis sama dengan aplikasi lama. Semua alur kerja, semua nama menu, semua laporan — sama. Karyawan tidak perlu belajar ulang. Operasional harian tidak boleh terganggu.

Di atas fondasi itu, kami tambahkan perbaikan: invoice 1 langkah, catatan aktivitas, pembagian peran, Bahasa Indonesia, dan koneksi ke SFA vendor.

Operasional Tidak Terganggu
Semua alur kerja, menu, dan laporan persis sama dengan aplikasi lama. Karyawan tidak perlu belajar ulang.
Fondasi Baru & Tahan Lama
Teknologi modern yang bisa dirawat 10+ tahun ke depan. Aman, cepat, dan bisa diakses dari mana saja.
Invoice 1 Langkah
Dari 3 langkah jadi 1 langkah — baik untuk membuat maupun merevisi invoice
Catatan Aktivitas & Peran
Setiap perubahan data tercatat. Setiap jabatan hanya lihat halaman sesuai tugasnya.

2 Masalah yang Ingin Diselesaikan

# Masalah Yang Terdampak Tingkat
1 Windows XP — sudah tidak ada update keamanan, rawan diserang Semua karyawan Kritis
2 Teknologi VB lama — tidak bisa diupdate, developer sulit memperbaiki IT / Developer Kritis
3 Bikin invoice 3 langkah; revisi harus ulang dari awal Sales, Finance Tinggi
4 Tidak ada catatan siapa mengubah data — tidak bisa lacak SuperAdmin, Manajemen Tinggi
5 Semua orang bisa lihat semua data — tidak ada batasan peran Manajemen Sedang
6 Hanya satu bahasa — tidak mendukung Bahasa Indonesia Karyawan Sedang
7 Data menumpuk tanpa arsip — aplikasi makin lambat Semua pengguna Sedang
8 Tidak ada pemantauan error — masalah tidak ketahuan sampai user lapor Developer Rendah

3 Apa yang Akan Diperbaiki?

Aplikasi Lama
Invoice dibuat dalam 3 langkah terpisah
Kalau ada revisi, harus ulang semua langkah dari awal
Aplikasi Baru
Invoice dibuat dalam 1 langkah
Revisi juga 1 langkah — lebih cepat, lebih sedikit kesalahan
Aplikasi Lama
Tidak ada catatan aktivitas
Tidak tahu siapa yang membuat, mengubah, atau menghapus data
Aplikasi Baru
Semua aktivitas tercatat otomatis
Setiap buat, ubah, hapus data — tercatat siapa, apa, jam berapa
Aplikasi Lama
Semua orang lihat semua data
Sales bisa lihat data keuangan, Finance bisa akses data gudang
Aplikasi Baru
Akses dibatasi sesuai jabatan
Sales hanya lihat halaman penjualan. Finance hanya lihat halaman keuangan. Admin bisa lihat lebih banyak.
Aplikasi Lama
Invoice dari SFA vendor tidak terhubung
Harus input ulang manual
Aplikasi Baru
Invoice dari SFA vendor otomatis masuk
Saat sales vendor buat invoice di SFA mereka, otomatis tercatat di Jaza Venus
Aplikasi Lama
Hanya Bahasa Inggris, hanya komputer kantor
Aplikasi Baru
Bahasa Indonesia + Inggris, bisa dibuka dari mana saja
Laptop, tablet, HP — selama ada internet

Sistem & Fitur Tambahan

  • Login / Logout dengan password dan kode keamanan (MFA untuk SuperAdmin)
  • Ganti password
  • Atur peran pengguna (siapa boleh buka halaman apa)
  • Catatan aktivitas — semua buat/ubah/hapus data tercatat
  • Pemantauan error — untuk tim IT
  • Bahasa Indonesia + Bahasa Inggris — bisa pilih kapan saja
  • Dashboard ringkasan harian
  • Data 5 tahun terakhir aktif, data lama diarsipkan (masih bisa dibuka)
  • Pindahkan seluruh data dari aplikasi lama → aplikasi baru

8 Hal yang Perlu Diperhatikan

Risiko Dampak Solusi Kami
Data lama sangat besar Pemindahan data bisa lambat Uji coba pindah data dulu. Data > 5 tahun langsung diarsipkan saat pemindahan.
Karyawan terbiasa aplikasi lama 10+ tahun Bisa bingung kalau tampilan beda jauh Tampilan dan menu dibuat semirip mungkin. Nama-nama menu sama persis dengan aplikasi lama.
Beberapa cabang internet lambat Halaman lama terbuka Aplikasi dibuat ringan. Data ditampilkan bertahap, tidak sekaligus.
Data keuangan sensitif Risiko kecurangan internal Setiap aktivitas tercatat. Akses dibatasi per jabatan. Tidak bisa hapus catatan aktivitas.
Vendor SFA punya format berbeda Invoice bisa gagal masuk Format invoice diseragamkan. Semua kiriman dicatat — kalau gagal, langsung ketahuan.

Setelah dokumen ini disepakati, kami akan mulai membuat tampilan demo (POC) untuk dilihat dan diberi masukan.
